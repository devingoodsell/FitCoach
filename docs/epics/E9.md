# E9 — Training Locations & Equipment

**Goal:** Multiple locations with distinct equipment, and a switchable current context. **Wave 1.**
Depends on: E3. Parallel with: E2, E4, E7, E11, E13.

Stories: E9-S1 manage multiple locations, E9-S2 switch current context. (Both `[P0]`.)

## PRs

### E9-PR1 — Locations schema & contract `[backend]` `[contract]` — *E9-S1*

  API in `backend/api/`. Stored in Coach Memory (E3).

- Done: schema + contract fixed.

- Tests: switching context changes assembled equipment; emits a re-plan trigger.
- Done: planning uses today's equipment.

### E9-PR3 — Locations & context UI `[android]` — *E9-S1, E9-S2*
- `feature/location`: create/edit locations with equipment; switch current context.
- Tests: location CRUD; context switch reflects in state.
- Done: user manages locations and switches context.
