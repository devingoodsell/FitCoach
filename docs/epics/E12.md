# E12 — Offline, Sync & Caching

**Goal:** Training works without connectivity; data syncs reliably when it returns. **Wave 3.**
Depends on: E5 (caching), E3 (sync). Parallel with: E6, E14.

Stories: E12-S1 run a session fully offline, E12-S2 sync logged data when back online. (Both `[P0]`.)

## PRs

### E12-PR1 — Offline session guarantee `[android]` — *E12-S1*
- Ensure the cached session (E5-PR5) + all in-session features (logging, timers, rest countdowns,
  autoregulation) run **end-to-end with no connectivity**. Audit the session path for any network dependency.
- Tests: full session completes with the network disabled (instrumented test in airplane mode).
- Done: a dead signal never blocks training.

### E12-PR2 — Offline write queue & idempotent sync `[android]`+`[backend]` — *E12-S2*
- `core/db` durable queue for offline-logged work; on reconnect, flush to the backend. **Idempotent**
  (stable client-generated IDs) so reconnecting never duplicates records. Backend upserts on those IDs
  (ties to E3-PR5).
- Tests: queued work syncs on reconnect; replaying the same payload does not duplicate; partial-failure retries safely.
- Done: nothing logged offline is lost or duplicated.

## Definition of done (epic)
A generated session runs and logs fully offline, then syncs idempotently on reconnect with no loss
or duplication.

## Status — ✅ DONE (Wave 3, `wave3-mvp`)
- **E12-PR1:** the session path was audited — its only network touch is the single generate() at
  "Start workout" (pre-loop, falls back to cache); the run/log/rest/timer/complete loop reads the
  Room cache and writes locally. Proven by an instrumented test that drives the player with the
  emulator in real airplane mode.
- **E12-PR2:** `workout_outbox` (Room) is a durable queue keyed by the stable `client_session_id`;
  `WorkoutSyncManager` flushes to `POST /workouts`, removing accepted rows and keeping failed ones
  for safe retry. Idempotent end-to-end (the backend upserts on the key). The OpenAPI `data` shape
  was documented additively as `WorkoutLogData` (backend still stores it opaquely). TDD on
  idempotency + partial-failure; instrumented DAO + 2→3 migration green.
