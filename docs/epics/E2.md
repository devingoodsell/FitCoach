# E2 — Adaptive Onboarding & User Model

**Goal:** Capture a usable user model quickly, expanding only when relevant; every field editable
forever. **Wave 1.** Depends on: E3. Parallel with: E4, E7, E9, E11, E13.

Stories: E2-S1 adaptive onboarding, E2-S2 profile/physiology, E2-S3 experience level,
E2-S4 goal sliders, E2-S5 schedule, E2-S6 `[P1]` exercise/equipment prefs, E2-S7 dietary prefs,
E2-S8 `[P1]` healthy-aging emphases defaults.

## PRs

### E2-PR1 — User-model contract & validation `[backend]` `[contract]` — *E2-S1*
- Extend `backend/api/` + `internal/onboarding` validation for the minimum-to-generate fields
  (age/DOB, sex, experience, goal weights, schedule, ≥1 location w/ equipment). Persists via E3.
- Tests: required-field validation; partial save allowed for optional sections.
- Done: contract for the user model fixed; writes land in Coach Memory.

### E2-PR2 — Profile & physiology `[backend]` — *E2-S2*

- Tests: age derived from DOB; timestamped weight stored.
- Done: physiology captured for programming.

### E2-PR3 — Experience & benchmark lifts `[backend]` — *E2-S3*

## Definition of done (epic)
Onboarding writes a complete user model fast, expands only when relevant, communicates time, and
every field is captured in editable Coach Memory (editing UI itself is E14).
