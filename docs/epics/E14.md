# E14 — Settings & Profile Management

**Goal:** Everything captured at onboarding is editable forever. **Wave 3.** Depends on: E2, E3.
Parallel with: E6, E12. Mostly `[android]` (backend write paths already exist from E2/E3/E9/E7).

Stories: E14-S1 edit all user-model fields, E14-S2 `[P1]` review disclaimers & consent state.

## PRs

### E14-PR1 — Settings hub & profile/goals/schedule editing `[android]` — *E14-S1*
- `feature/settings`: edit profile/physiology, goal weights, and schedule; edits persist to Coach
  Memory (E2/E3) and take effect on the next planning call.
- Tests: edits round-trip to memory; changed fields reflected on next assembly.
- Done: core user-model fields editable post-onboarding.

### E14-PR2 — Edit preferences, locations, diet, aging emphases `[android]` — *E14-S1*
- Editing surfaces for exercise/equipment prefs, locations & current context (reuse `feature/location`),
  diet preferences, and aging emphases.
- Tests: each section edits and persists; hard-avoids/hard-diet-constraints preserved.
- Done: every onboarding field is editable.
- **Carryover from E2 (E2-S8):** the onboarding wizard does **not** surface aging-emphasis
  *adjuster sliders* — the backend defaults them from age (`onboarding.DefaultAgingEmphases`) and
  stores them under the `profile` memory section as `aging_emphases`. Build the 4 sliders
  (bone/balance, joint/tendon, vo2max, cardio base) here so users can override the defaults; persist
  back via the profile write path. This is the only P1 onboarding field with no edit UI yet.

### E14-PR3 — Disclaimers & consent review `[android]`+`[backend]` `[P1]` — *E14-S2*
- Show current consent state + the medical disclaimer (E13-PR1 text); **revoke health-data consent**
  → disables ingestion (E4) and falls back to manual mode.
- Tests: consent state shown; revocation disables ingestion and switches to manual.
- Done: user can review what they agreed to and revoke consent.

## Definition of done (epic)
Every onboarding field is editable in Settings with edits persisting to Coach Memory; consent state
and disclaimers are reviewable and health-data consent is revocable.
