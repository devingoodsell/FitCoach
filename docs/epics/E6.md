# E6 — In-Session Experience

**Goal:** A clean execution surface: start, log, time, and rest with countdowns. **Wave 3.**
Depends on: E5 (UI scaffold can start on the E5-PR1 sample). Parallel with: E12, E14. Mostly `[android]`.

Stories: E6-S1 start/run, E6-S2 log reps/weight, E6-S3 rest countdowns, E6-S4 timed-exercise
timers, E6-S5 mark complete. (All `[P0]`.) **Every feature must work offline.**

## PRs

### E6-PR1 — Session player scaffold `[android]` — *E6-S1*
- `feature/session`: load a cached session (E5-PR5); ordered list of warmup/main/accessory/aging;
  current exercise with target reps/load/rest clearly shown. Built against the E5-PR1 sample first.
- Tests: ordered blocks render; current-set details shown; works from cache offline.
- Done: user starts and moves through the session set by set.

### E6-PR2 — Log reps & weight `[android]` — *E6-S2*
- Per-set reps/weight entry defaulting to prescribed targets; logged values feed autoregulation
  (E5-PR6) and Coach Memory (E3-PR5).
- Tests: defaults prefilled; edits captured; values flow to autoregulation + log queue.
- Done: real performance recorded per set.

### E6-PR3 — Rest countdown timers `[android]` — *E6-S3*
- Auto-start a countdown for the prescribed rest after logging a set (where appropriate);
  skip/extend/pause; **audible + haptic cue at zero.** Offline.
- Tests: countdown starts/skip/extend/pause; cue fires at zero.
- Done: prescribed rests are enforced with cues.

### E6-PR4 — Timers for timed exercises `[android]` — *E6-S4*
- Count-up/down timer with start/stop for timed work (e.g. planks); completed duration logged like
  reps/weight.
- Tests: timer runs; completed duration logged.
- Done: timed work is timed and recorded.

### E6-PR5 — Mark session complete (incl. partial) `[android]` — *E6-S5*
- Finish + save writes all logged work to Coach Memory (E3-PR5); **partial sessions save with skipped
  work recorded as skipped.** Queues for sync offline (E12).
- Tests: complete writes full log; partial records skips; offline completion queues.
- Done: sessions (full or partial) are recorded and inform future plans.

## Definition of done (epic)
A full session runs end-to-end — start, log reps/weight, rest countdowns with cues, timed-exercise
timers, and completion (incl. partial) — entirely offline, writing results to Coach Memory.
