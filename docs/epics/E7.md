# E7 — Injury & Condition Management

**Goal:** Injuries as first-class, structured memory objects that constrain the plan, with a
deterministic safety net. **Wave 1.** Depends on: E3 (+ E13 for safety). Parallel with: E2, E4, E9, E11.

> **Agree the contraindication rule shape with E13 before coding** (shared interface).

Stories: E7-S1 natural-language injury entry, E7-S2 lifecycle, E7-S3 injuries constrain the plan,
E7-S4 deterministic safety validation, E7-S5 `[P1]` identification assist.

## PRs

### E7-PR1 — Injury schema & contract `[backend]` `[contract]` — *E7-S1, E7-S2*
- Migration for `injuries` (region, status, severity, aggravating_movements, onset_date, notes,
  timestamps; status ∈ active-flare/managed/recurring-but-fine/resolved). API in `backend/api/`.
- Tests: CRUD persists; status enum enforced; entry stored with its date.
- Done: structured injury model + contract fixed.

### E7-PR2 — Natural-language parse to structure `[backend]` — *E7-S1*
- Parse a freeform description into the structured slots (LLM-assisted, server-side, via E15 seam),
  returning a draft for the **user to review/correct before saving**.
- Tests: representative inputs map to expected slots; low-confidence fields flagged for review.
- Done: freeform → reviewable structured draft.

### E7-PR3 — Lifecycle & re-plan trigger `[backend]` — *E7-S2*
- Add/edit/remove; a new/changed injury **immediately triggers re-planning** (handshake with E5-S5).
- Tests: status change emits a re-plan trigger; edits update memory.
- Done: status changes alter programming behavior right away.

### E7-PR4 — Contraindication injection into planning `[backend]` — *E7-S3*
- Active/managed conditions inject contraindications + substitution preferences into E3 prompt
  assembly; reasoning later notes substitutions (E5-S2).
- Tests: active injury adds the expected contraindications to the assembled payload.
- Done: planning receives injury constraints.

### E7-PR5 — Safety validation hook (with E13) `[backend]` — *E7-S4*
- In `internal/safety`: validate a generated plan against **hard injury contraindications** server-side,
  between model and user; violating plans are corrected or regenerated, **never shown as-is**;
  outcomes logged (E15-S3/E13). Shares the rule interface agreed with E13.
- Tests (TDD, mandatory): a plan violating a contraindication is corrected/rejected; a clean plan passes;
  outcome logged.
- Done: unsafe plans can't reach the user regardless of model output.

### E7-PR6 — Injury entry & lifecycle UI `[android]` — *E7-S1, E7-S2*
- `feature/injury`: freeform box → review/correct parsed result → save; manage status over time.
- Tests: review-before-save enforced; status edits persist.
- Done: user describes an injury in NL and manages it.

### E7-PR7 — Identification assist `[android]`+`[backend]` `[P1]` — *E7-S5*
- Guided Q&A to characterize an unknown issue, framed with explicit **"not a diagnosis, consult a
  clinician"** disclaimers (E13-S1); result saved as a normal injury the user confirms.
- Tests: disclaimer present throughout; output becomes a confirmable injury entry.
- Done: assist flow respects the same safety/disclaimer language.

## Definition of done (epic)
Injuries parse to structured memory, have an editable lifecycle that triggers re-planning, inject
contraindications into planning, and a deterministic safety layer corrects/rejects unsafe plans.
