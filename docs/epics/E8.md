# E8 — Healthy-Aging / Longevity Programming

**Goal:** Make healthspan work a first-class part of every session, not an afterthought. **Wave 2 —
built together with E5**, not as a separate service. Depends on: E5, E2.

Stories: E8-S1 aging/mobility block in every session, E8-S2 age-appropriate programming. (Both `[P0]`.)

## PRs

### E8-PR1 — Aging block in generation `[backend]` — *E8-S1*
- Extend the E5 generation prompt/parse so **every** session includes a healthspan block targeting
  the user's emphases (bone/balance, joint/tendon, VO2max/high-intensity, cardio base) from E2-PR7
  settings + age. The block **respects active injuries and the safety layer** (E7/E13).
- Tests: generated session always contains an aging block matching the user's emphases; block passes safety.
- Done: aging/mobility block present in every session.

### E8-PR2 — Age-appropriate programming `[backend]` — *E8-S2*
- Age + aging emphases measurably influence volume, intensity, recovery, and exercise selection in
  generation; reasoning **surfaces at least one age-aware choice** when relevant.
- Tests: an older vs. younger profile (same other inputs) yields measurably different programming;
  reasoning notes an age-aware choice.
- Done: programming differs by age, with a visible rationale.

## Definition of done (epic)
Every generated session carries an emphasis-targeted, injury-respecting aging block, and programming
demonstrably adapts to age with at least one age-aware reasoning note.
