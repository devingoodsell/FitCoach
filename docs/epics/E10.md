# E10 — Classes (Equinox) — V1.1 (deferred)

**Goal:** Ingest the user's gym class schedule and let the engine slot classes around training when
they serve the plan better than another lift. **Wave 4 / V1.1 — not in MVP.** Depends on: E5.

> No MVP stories. Equinox classes are priority #4 in the spec and explicitly deferred to V1.1 (see
> [`../user-stories/README.md`](../user-stories/README.md) and `v1.1-full-release.md`). This file is a
> placeholder so the epic map is complete; flesh out PRs when V1.1 planning starts.

## Likely PR shape (to refine for V1.1)
- `[backend]` Class schedule ingestion — user-entered, with PDF/screenshot upload as a fallback
  (no public gym API; see spec §12 risks). Parse into structured class entries.
- `[backend]` Class slotting in generation — engine recommends a class in place of a lift when it
  better serves the plan; surfaced as the optional class suggestion in the E5 session output.
- `[android]` Class entry + upload UI and the in-session/plan class suggestion surface.

## Definition of done (V1.1)
Users can supply a weekly class schedule; the engine optionally recommends slotting a class around
training with plain-language reasoning, validated by the same safety layer.
