# E15 — Backend Platform & Observability

**Goal:** A secure, observable backend that holds the Claude key, assembles prompts, calls Claude,
validates, and persists. **Wave 0 (foundational).** Depends on: nothing. Gates: everything.

Stories: E15-S1 (server-side orchestration), E15-S2 (persist profile/memory/history),
E15-S3 `[P1]` (generation & safety logging).

> This epic delivers the *platform*; the actual Claude call lives in E5 and account data models in
> E1/E3. Here we build the skeleton those plug into.

## PRs

### E15-PR1 — Repo & Go module skeleton `[infra]`
- `backend/go.mod`, `cmd/server/main.go` (boots, reads config, starts HTTP, graceful shutdown),
  `internal/platform/httpx` router + JSON/error helpers, a `GET /healthz` endpoint.
- Tests: `/healthz` returns 200; server starts/stops cleanly.
- Done: `go run ./cmd/server` serves `/healthz`; `go vet`/`gofmt` clean.

### E15-PR2 — Config & secrets loading `[infra]` — *advances E15-S1*
- `internal/platform/config`: load from env (`ANTHROPIC_API_KEY`, `MYSQL_DSN`, `JWT_SIGNING_KEY`,
  TTLs, `HTTP_ADDR`, `LOG_LEVEL`). Fail fast on missing required values. `.env.example` documents all.
- Tests: required-var validation; key never logged (assert redaction).
- Done: key is loadable server-side only; absent from any log line.

### E15-PR3 — MySQL connection & migration runner `[infra]` — *advances E15-S2*
- `internal/platform/db`: pooled `*sql.DB`, tx helper, ordered migration runner reading
  `backend/migrations/`. First migration creates the migrations-tracking table.
- Tests: runner applies migrations idempotently against a test DB (or sqlmock for the runner logic).
- Done: `migrate` runs on boot or via subcommand; re-running is a no-op.

### E15-PR4 — Structured logging with redaction `[infra]` — *advances E15-S3*
- `internal/platform/logging`: leveled JSON logger; helpers that redact secrets & PII; request-ID
  middleware so every request log correlates.
- Tests: redaction of known-sensitive keys; level filtering.
- Done: requests log method/path/status/latency/request-id; no secrets.

### E15-PR5 — CI pipeline `[infra]`
- CI config: `gofmt -l`, `go vet`, `go test ./...` for backend; lint + unit tests for Android.
- Done: CI green on a trivial PR; fails on a formatting/test regression.

### E15-PR6 — Generation & safety event logging `[backend]` `[P1]` — *E15-S3*
- A small `events` writer the `coaching` (E5) and `safety` (E13/E7-S4) packages call to record
  generation inputs (redacted), latency, outcome, and safety pass/correct/regenerate. Access-controlled.
- Tests: an event is persisted with secrets stripped; redaction holds for prompt contents.
- Done: E5/E13 have a logging seam to call (even if those epics land later).

## Definition of done (epic, MVP slice)
Skeleton boots, config/secrets load safely, migrations run, structured logging redacts, CI green,
and a logging seam exists for generation/safety events.
