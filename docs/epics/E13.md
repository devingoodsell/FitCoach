# E13 — Safety & Compliance Layer

**Goal:** Honest, safe guidance with hard limits and clear disclaimers. **Wave 1** (deterministic;
coordinate with E7). Depends on: nothing hard (rule shape coordinated with E7). Parallel with: E2, E4, E9, E11.

> The injury-contraindication half of the safety layer is built jointly with **E7-PR5**; this epic
> owns disclaimers and the **load/volume bounds** half. **Agree the rule interface with E7 first.**

Stories: E13-S1 disclaimers wherever the body is involved, E13-S2 enforce load/volume bounds. (Both `[P0]`.)

## PRs

### E13-PR1 — Central disclaimer text store `[backend]` `[contract]` — *E13-S1*
- A single source of disclaimer/medical-language strings, versioned (ties to E1 consent version),
  served to the client and retrievable in Settings (E14-S2).
- Tests: text is centralized (no inline duplicates); version surfaces with consent.
- Done: one place owns disclaimer copy.

### E13-PR2 — Disclaimer surfacing (client) `[android]` — *E13-S1*
- `core/designsystem` disclaimer component shown at onboarding, injury flows, readiness, and diet
  guidance; full text retrievable in Settings.
- Tests: disclaimer renders in each required surface; Settings shows the current text.
- Done: "guidance, not medical advice" appears wherever health is involved.

### E13-PR3 — Load/volume bounds engine `[backend]` — *E13-S2*
- In `internal/safety`: configurable bounds; cap load/volume of a generated plan **before delivery**,
  independent of the model; **log bound violations** (E15-S3). Shares the safety entrypoint with E7-PR5.
- Tests (TDD, mandatory): an over-bound plan is capped; a within-bounds plan passes unchanged;
  violations logged; bounds are config-driven.
- Done: no session exceeds configured safe limits regardless of model output.

### E13-PR4 — Unified safety pipeline `[backend]` — *E13-S2 + E7-S4*
- Compose the contraindication check (E7-PR5) and bounds check (E13-PR3) into one
  `safety.Validate(plan, memory)` the engine (E5) calls; returns pass / corrected / regenerate.
- Tests: combined cases (injury + bound violations) resolve correctly; deterministic.
- Done: E5 has a single safety entrypoint.

## Definition of done (epic)
Disclaimers are centrally managed and surfaced everywhere health is involved; a deterministic safety
pipeline (contraindications + load/volume bounds) validates every plan before delivery and logs outcomes.
